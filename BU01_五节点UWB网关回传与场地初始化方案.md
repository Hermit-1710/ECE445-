# BU01 五节点 UWB 网关回传与场地初始化方案

## 1. 最终敲定方案

本项目使用 5 块 BU01：

| 节点 | 角色 | 是否接电脑 | 作用 |
| --- | --- | --- | --- |
| A0 | 网关锚点 / 高位 Anchor 0 | 是 | 放桌面或支架上，接收标签结果并通过串口传给电脑，同时提供高度约束 |
| A1 | 地面 Anchor 1 | 否 | 固定在地面，响应测距 |
| A2 | 地面 Anchor 2 | 否 | 固定在地面，响应测距 |
| A3 | 地面 Anchor 3 | 否 | 固定在地面，响应测距 |
| TAG | 球内标签 | 否 | 轮询 4 个锚点测距，并通过 UWB 把结果发给 A0 |

核心链路：

```text
TAG --UWB TWR--> A0/A1/A2/A3
TAG --UWB 数据帧--> A0
A0 --UART/USB--> 电脑
电脑 --定位解算/轨迹显示--> 实时界面
```

这个方案满足：

1. 只有 1 个锚点连接电脑。
2. 标签不需要连接电脑。
3. 不额外增加蓝牙、WiFi、LoRa 等通信模块。
4. 所有无线通信都使用 BU01 自带 UWB 天线完成。
5. 锚点之间不需要有线同步，因为主路线仍然是 TWR。

## 2. 为什么不能让普通锚点直接“自动把距离传电脑”

TWR 测距中，距离通常由发起方根据本地时间戳和对方回传时间戳计算。

本方案让 TAG 作为发起方：

```text
TAG -> Anchor i: POLL
Anchor i -> TAG: RESP
TAG: 计算 d_i
```

因此距离 `d0, d1, d2, d3` 首先出现在 TAG 端。由于 TAG 在球内，不能接电脑，所以必须增加一步：

```text
TAG -> A0: 上传一帧距离数据
```

A0 收到后再通过串口发给电脑。

## 3. 运行阶段通信流程

### 3.1 单帧定位周期

每一帧定位数据包含以下步骤：

1. TAG 与 A0 进行 TWR，得到 `d0`
2. TAG 与 A1 进行 TWR，得到 `d1`
3. TAG 与 A2 进行 TWR，得到 `d2`
4. TAG 与 A3 进行 TWR，得到 `d3`
5. TAG 将 `{seq, d0, d1, d2, d3, status}` 通过 UWB 数据帧发送给 A0
6. A0 通过串口发给电脑
7. 电脑根据 4 个距离和 4 个锚点坐标解算 TAG 的三维坐标

电脑收到的数据建议格式：

```text
RANGE4,seq,d0_cm,d1_cm,d2_cm,d3_cm,status,timestamp_ms
```

示例：

```text
RANGE4,125,230,512,640,455,0F,382910
```

## 4. 坐标系如何定义

系统坐标不是绝对 GPS 坐标，而是相对于 4 个锚点建立的场地坐标系。

根据你的实际场景，推荐改成：

1. A1 是地面原点：`A1 = (0, 0, 0)`
2. A2 在地面 X 轴正方向：`A2 = (d12, 0, 0)`
3. A3 位于地面 X-Y 平面，且 `y > 0`
4. A0 放桌面或支架上，位于三维空间，且 `z > 0`

也就是说，后续算出来的球坐标 `(x, y, z)` 是：

```text
相对于 A1 原点、A1->A2 为 X 轴、A1/A2/A3 所在地面为 XY 平面的坐标
```

A0 虽然接电脑，但不再作为原点，而是作为高位锚点提供 Z 方向约束。这样图形化界面中的 `z=0` 就天然表示地面，更适合做落点判断。

## 5. 初始化阶段锚点怎么摆

### 5.1 推荐摆放

为了让三维定位稳定，4 个锚点不能全部共线，也不建议全部完全等高。

推荐：

| 锚点 | 位置建议 |
| --- | --- |
| A0 | 接电脑，放桌面或支架上，作为高位网关锚点 |
| A1 | 放地面，作为场地坐标原点 |
| A2 | 放地面，沿 X 方向和 A1 拉开 |
| A3 | 放地面另一侧，和 A1/A2 构成较大三角形 |

理想几何形状：

```text
A3
|
|
A1 -------- A2

A0 放桌面或支架上，位于地面三角形附近或一侧
```

### 5.2 推荐距离

实验阶段可以先小范围：

```text
A1-A2: 3 m 到 5 m
A1-A3: 3 m 到 5 m
A2-A3: 4 m 到 7 m
A0: 放在桌面或支架上，高度建议 0.8 m 到 1.5 m
```

真实场地阶段再扩大到 10 m 以上。

### 5.3 摆放原则

1. A1、A2、A3 在地面构成尽量大的三角形。
2. A0 放高，避免四个锚点共面。
3. A0 必须靠近电脑，因为 A0 负责串口回传。
4. A1/A2/A3 固定后不要移动，否则需要重新初始化。

## 6. 初始化阶段如何获得锚点相对距离

初始化阶段使用“锚点间 TWR 自标定”。

需要测量 6 条锚点间距离：

```text
d01 = distance(A0, A1)
d02 = distance(A0, A2)
d03 = distance(A0, A3)
d12 = distance(A1, A2)
d13 = distance(A1, A3)
d23 = distance(A2, A3)
```

### 6.1 初始化通信方式

A0 作为初始化控制器，向其他锚点发送命令：

```text
A0 -> A1: 你和 A2 测距
A1 -> A2: TWR
A1 -> A0: 回传 d12
```

类似地获得 `d13`、`d23`。

最小实现可以先分阶段做：

1. A0 分别和 A1/A2/A3 测距，得到 `d01,d02,d03`
2. 临时切换程序或命令，让 A1-A2、A1-A3、A2-A3 分别测距
3. 所有距离最终都通过 A0 串口传给电脑

工程成熟后再做成一键初始化。

### 6.2 锚点坐标计算

有 6 条边长后，可以自动建立相对坐标。

根据新的坐标系，设：

```text
A1 = (0, 0, 0)
A2 = (d12, 0, 0)
```

A3：

```text
x3 = (d13^2 + d12^2 - d23^2) / (2*d12)
y3 = sqrt(d13^2 - x3^2)
z3 = 0
```

A0：

```text
x0 = (d01^2 + d12^2 - d02^2) / (2*d12)
y0 = (d01^2 - d03^2 - x0^2 + (x0-x3)^2 + y3^2) / (2*y3)
z0 = sqrt(d01^2 - x0^2 - y0^2)
```

默认取 `z0 > 0`，表示 A0 在地面平面上方。

这样电脑界面就能自动得到：

```text
A1 = (0, 0, 0)
A2 = (x2, y2, z2)
A3 = (x3, y3, z3)
A0 = (x0, y0, z0)
```

## 7. 标签三维定位怎么算

运行时电脑收到：

```text
d0,d1,d2,d3
```

已知：

```text
A0(x0,y0,z0)
A1(x1,y1,z1)
A2(x2,y2,z2)
A3(x3,y3,z3)
```

求 TAG：

```text
P = (x, y, z)
```

满足：

```text
|P - A0| = d0
|P - A1| = d1
|P - A2| = d2
|P - A3| = d3
```

电脑端使用最小二乘求解：

```text
min sum((|P - Ai| - di)^2)
```

连续帧再做中值滤波或卡尔曼滤波，得到平滑轨迹。

## 8. 图形化界面初始化流程

图形界面建议分为两个模式：

1. 场地初始化模式
2. 实时轨迹模式

### 8.1 场地初始化模式

界面流程：

1. 点击“扫描节点”
2. A0 发现 A1/A2/A3/TAG
3. 点击“锚点自标定”
4. 系统测量 6 条锚点间距离
5. 电脑计算 A0/A1/A2/A3 相对坐标
6. 界面显示锚点布局图
7. 用户确认 A3 在上方，必要时手动翻转 Z 方向
8. 保存场地配置文件

配置文件示例：

```json
{
  "anchors": {
    "A1": [0.0, 0.0, 0.0],
    "A2": [4.0, 0.0, 0.0],
    "A3": [1.2, 3.8, 0.0],
    "A0": [2.0, 1.5, 1.0]
  },
  "unit": "m"
}
```

### 8.2 实时轨迹模式

界面流程：

1. 打开 A0 串口
2. 接收 `RANGE4` 数据
3. 解算 TAG 三维坐标
4. 显示球的实时位置点
5. 连续点形成轨迹
6. 输出速度、最高点、落点和出界判断

## 9. 固件角色设计

5 块 BU01 固件建议分为三类：

| 固件 | 节点 | 功能 |
| --- | --- | --- |
| Gateway Anchor | A0 | 接电脑、参与 TWR、接收 TAG 数据、输出串口 |
| Normal Anchor | A1/A2/A3 | 响应 TWR、执行初始化命令 |
| Tag | TAG | 轮询测距、汇总距离、UWB 回传 A0 |

### 9.1 A0 网关锚点

A0 需要支持：

1. 与 TAG 测距。
2. 接收 TAG 的 `RANGE4` 数据帧。
3. 通过串口输出给电脑。
4. 初始化阶段控制锚点间测距。

### 9.2 普通锚点 A1/A2/A3

普通锚点需要支持：

1. 响应 TAG 的 TWR 请求。
2. 初始化阶段响应 A0 的控制命令。
3. 必要时主动和其他锚点测距，并把结果通过 UWB 回传 A0。

### 9.3 标签 TAG

TAG 需要支持：

1. 依次对 A0/A1/A2/A3 测距。
2. 将四个距离打包成一帧。
3. 通过 UWB 发给 A0。

## 10. 推荐开发阶段

### 阶段 A：当前已完成

1. 单板 SPI/DEV_ID/IRQ/TX 验证。
2. 1 标签 + 1 锚点 SS-TWR 测距。
3. 标签端串口实时输出距离。

### 阶段 B：下一步

目标：实现 `TAG -> A0 -> 电脑`。

内容：

1. TAG 先只和 A1 测距。
2. TAG 不再通过串口输出，而是把距离用 UWB 数据帧发给 A0。
3. A0 接收后通过串口输出：

```text
RELAY,seq,d1_cm,raw_cm,status
```

验收：

```text
TAG 不接电脑，电脑仍然能从 A0 串口看到实时距离。
```

### 阶段 C：四锚点轮询

目标：实现：

```text
TAG -> A0/A1/A2/A3 测距
TAG -> A0 回传 d0,d1,d2,d3
A0 -> 电脑输出 RANGE4
```

验收：

```text
RANGE4,seq,d0,d1,d2,d3,status
```

### 阶段 D：锚点自标定

目标：

1. 测量 6 条锚点间距离。
2. 自动生成相对坐标。
3. 保存场地配置。

### 阶段 E：三维轨迹

目标：

1. 根据 `RANGE4` 解算 TAG 坐标。
2. 实时显示轨迹。
3. 做速度和落点判断。

## 11. 关键结论

最终推荐方案：

```text
5 块 BU01 = 4 锚点 + 1 标签
A0 接电脑作为网关
TAG 轮询测距并通过 UWB 回传 A0
A0 串口输出给电脑
电脑完成坐标解算和轨迹显示
```

坐标系定义：

```text
A1 为地面原点
A1 -> A2 为 X 轴
A1/A2/A3 定义地面 XY 平面
A0 放高并定义 Z 方向
```

初始化方式：

```text
先测 6 条锚点间距离
再由电脑自动计算 4 个锚点相对坐标
最后进入实时轨迹模式
```

这套方案不需要额外无线模块，不需要锚点有线同步，符合“只使用 BU01 自带 UWB 天线功能”的要求。
